# Iterative-Probe vs AutoGPT 差距分析与升级计划

## 1. 当前 iterative-probe 能力总结

### 核心定位
专用于 C++/UE 项目的渐进式迭代验证策略，用于"边界模糊、边做边发现"的复杂系统集成场景。

### 已有能力

| 能力 | 描述 |
|------|------|
| **迭代循环** | 探测 → 分析 → 修复 → 验证 → 循环 |
| **自驱模式** | 强制自主运行到终止条件，不中途停止 |
| **并行探测** | 支持多条独立链路同时探测 |
| **日志记录** | 完整的 probe_log.md 过程追溯 |
| **子 Agent 协作** | 主 Agent 决策 + 子 Agent 探测 |
| **用户协助机制** | 钉钉通知 + AskUserQuestion |
| **终止条件** | 目标达成 / 硬阻塞 / 需求不明确 / 迭代上限 |

### 架构模式
```
主 Agent（决策层）
    │
    ├── 定义探测目标
    ├── 启动子 Agent
    ├── 分析结果
    └── 决策：修复 / 深入 / 完成
         │
         ▼
子 Agent（执行层）
    │
    └── 执行探测任务，返回结果
```

#### 2.1 长期记忆系统 (Long-term Memory)

| 组件 | 功能 |
|------|------|

```python
# AutoGPT 记忆系统伪代码
class MemorySystem:
    def store(self, content, metadata):
        embedding = self.embed(content)
        self.vector_db.insert(embedding, content, metadata)

    def recall(self, query, k=5):
        embedding = self.embed(query)
        return self.vector_db.search(embedding, k)

    def summarize_old_memories(self):
        old = self.get_old_memories()
        summary = self.llm.summarize(old)
        self.replace_with_summary(old, summary)
```

## 3. 差距分析

### 3.1 能力对比矩阵

| 能力维度 | iterative-probe | AutoGPT | 差距等级 |
|----------|-----------------|---------|----------|
| **记忆系统** | ❌ 无 | ✅ 向量数据库 + 语义检索 | 🔴 严重 |
| **跨会话持久化** | ⚠️ 仅日志文件 | ✅ 完整持久化 | 🔴 严重 |
| **自主规划** | ⚠️ 手动定义目标 | ✅ 自动目标分解 | 🟡 中等 |
| **动态重规划** | ⚠️ 有限 | ✅ 完整支持 | 🟡 中等 |
| **自我评估** | ⚠️ 检查清单 | ✅ 系统化评估 | 🟡 中等 |
| **反思学习** | ❌ 无 | ✅ 经验提取 | 🔴 严重 |
| **工具系统** | ✅ 使用 Claude 工具 | ✅ 动态工具系统 | 🟢 接近 |
| **并行执行** | ✅ 支持 | ✅ 支持 | 🟢 接近 |
| **多 Agent** | ⚠️ 主从模式 | ✅ 对等协作 | 🟡 中等 |
| **自驱循环** | ✅ 强制自驱 | ✅ 完全自主 | 🟢 接近 |
| **日志追溯** | ✅ 完整日志 | ✅ 完整日志 | 🟢 接近 |
| **领域专用** | ✅ C++/UE 专用 | ⚠️ 通用 | 🟢 优势 |

### 3.2 核心差距详解

#### 差距 1：无长期记忆系统 🔴

**现状**：
- 仅有 `probe_log.md` 文件记录
- 无法语义检索历史经验
- 每次会话从零开始

**影响**：
- 重复犯同样的错误
- 无法积累领域知识
- 无法复用成功模式

#### 差距 2：无反思学习机制 🔴

**现状**：
- 有"自驱心态检查清单"但仅是提醒
- 无系统化的经验提取
- 无失败模式识别

**影响**：
- 无法从错误中学习
- 无法优化探测策略
- 无法识别反模式

#### 差距 3：规划能力有限 🟡

**现状**：
- 需要用户定义探测目标
- 目标分解依赖人工
- 无自动优先级排序

**影响**：
- 依赖用户指导
- 无法处理模糊目标
- 效率受限

#### 差距 4：多 Agent 协作简单 🟡

**现状**：
- 仅支持主从模式（主 Agent 决策，子 Agent 执行）
- 无 Agent 间对等协作
- 无共享知识库

**影响**：
- 无法处理需要多角色协作的复杂任务
- 知识无法在 Agent 间流动

#### 1.3 记忆检索机制

```markdown
## 检索触发时机

1. **探测开始前**：检索相关历史经验
2. **遇到问题时**：检索类似问题的解决方案
3. **修复完成后**：检索验证方法

## 检索方式

### 基于关键词
- 从探测目标提取关键词
- 匹配经验记录的标签和内容

### 基于领域
- 根据当前任务领域（GAS/DataAsset/...）
- 检索同领域的经验

### 基于问题类型
- 识别问题类型（配置缺失/代码错误/链路断裂）
- 检索同类型问题的解决方案
```

#### 1.4 实现方式

```markdown
## 新增 Skill 行为

### 探测开始时
1. 读取 memory/index.json
2. 根据探测目标检索相关经验
3. 在 probe_log.md 中记录"参考经验"

### 探测结束时
1. 提取本次探测的关键经验
2. 生成经验记录文件
3. 更新 index.json

### 检索伪代码
```
function recallExperiences(goal, domain):
    index = read("memory/index.json")
    keywords = extractKeywords(goal)

    candidates = []
    for exp in index.experiences:
        score = 0
        if exp.domain == domain: score += 2
        for kw in keywords:
            if kw in exp.tags: score += 1
            if kw in exp.title: score += 1
        if score > threshold:
            candidates.append((exp, score))

    return sortByScore(candidates)[:5]
```
```

### 4.3 Phase 2: 反思学习

**目标**：从每次探测中提取可复用的经验和模式

#### 2.1 反思触发点

| 触发点 | 反思内容 |
|--------|----------|
| 每轮迭代后 | 本轮尝试了什么？效果如何？ |
| 修复成功后 | 为什么这个方案有效？ |
| 修复失败后 | 为什么失败？下次如何避免？ |
| 探测完成后 | 整体策略是否最优？ |

#### 2.2 反思模板

```markdown
## 迭代反思 #{N}

### 尝试记录
| 方法 | 预期 | 实际 | 有效性 |
|------|------|------|--------|
| {方法1} | {预期} | {实际} | ✅/❌ |

### 成功因素分析
- 什么方法有效？为什么？
- 是否可以更早使用这个方法？

### 失败因素分析
- 什么方法无效？为什么？
- 是否有预警信号被忽略？

### 模式识别
- 这个问题属于什么类型？
- 是否见过类似问题？
- 解决方案是否可泛化？

### 策略调整
- 下一轮应该优先尝试什么？
- 应该避免什么？
```

### 4.5 Phase 4: 多 Agent 协作

**目标**：支持多个 Agent 对等协作，共享知识

#### 4.1 角色定义

```markdown
## Agent 角色

| 角色 | 职责 | 专长 |
|------|------|------|
| **探测者** | 执行探测任务 | 代码阅读、链路追踪 |
| **分析者** | 分析探测结果 | 问题分类、根因分析 |
| **修复者** | 执行修复任务 | 代码修改、配置调整 |
| **验证者** | 验证修复效果 | 测试、回归检查 |
```

## 7. 总结

### 当前 iterative-probe 的优势
- ✅ 强制自驱模式，不中途停止
- ✅ 完整的日志追溯
- ✅ 支持并行探测
