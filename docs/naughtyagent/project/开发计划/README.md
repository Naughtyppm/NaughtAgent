## 开发原则

1. **底层优先** - 先稳定核心引擎，再丰富上层功能
2. **独立单元** - 每个单元可独立开发、测试、交付
3. **依赖明确** - 标注前置依赖，避免阻塞
4. **可验证** - 每个单元有明确的完成标准
