## 1. Claude Code

**定位**：AI 编程助手，理解整个代码库，跨文件、跨工具完成开发任务。

### 1.1 运行环境（多端统一）

| 环境 | 说明 |
|------|------|
| **Terminal CLI** | 核心引擎，命令行直接操作 |
| **VS Code 插件** | inline diff、@-mention、plan review、对话历史 |
| **JetBrains 插件** | diff 查看、选区上下文共享 |
| **Desktop App** | 独立应用，可视化 diff、多会话并行、定时任务、云会话 |
| **Web** | 浏览器运行，无需本地环境，支持长任务和并行 |

**关键点**：所有端共享同一个底层引擎，CLAUDE.md、设置、MCP 服务器跨端通用。

### 1.2 核心功能

#### Agent Loop（智能循环）
- 流式输出 + 错误恢复 + 中断控制
- Extended Thinking（深度推理），默认开启
- 自适应推理：根据 effort level 动态分配 thinking tokens
- `ultrathink` 关键词触发单次深度推理

#### 工具系统
- 文件读写（Read/Write/Edit）
- Bash 命令执行
- 代码搜索（Grep/Glob）
- 笔记本操作（Notebook）
- Web 搜索

#### 子代理系统（Sub-agents）

这是 Claude Code 最重要的架构特性之一。

| 子代理 | 模型 | 用途 | 工具限制 |
|--------|------|------|----------|
| **Explore** | Haiku（快速） | 代码搜索、分析，只读 | 禁止 Write/Edit |
| **Plan** | 继承主会话 | Plan Mode 下的代码调研 | 只读 |
| **General-purpose** | 继承主会话 | 复杂多步任务，需读写 | 全部工具 |
| **Bash** | 继承 | 独立上下文执行命令 | Bash |

**自定义子代理**：
- 用 Markdown 文件 + YAML frontmatter 定义
- 可配置：model、tools、disallowedTools、permissionMode、maxTurns、hooks、memory、skills
- 作用域：项目级 `.claude/agents/`、用户级 `~/.claude/agents/`、CLI 参数 `--agents`
- **Worktree 隔离**：`isolation: worktree`，子代理在独立 git worktree 工作
- **持久记忆**：子代理可有自己的 memory 目录（user/project/local 三种作用域）
- **前台/后台**运行模式

#### 记忆系统（Memory）

**双层记忆**：
1. **CLAUDE.md 文件**（用户编写）
   - 项目级：`./CLAUDE.md` 或 `./.claude/CLAUDE.md`
   - 用户级：`~/.claude/CLAUDE.md`
   - 组织级：系统目录（管理员部署）
   - 支持 `@path` 导入外部文件
   - `.claude/rules/` 分模块组织，支持 `paths` frontmatter 按文件路径条件加载

2. **自动记忆**（Claude 自己写）
   - 存储在 `~/.claude/projects/<project>/memory/`
   - `MEMORY.md` 作为索引（前 200 行每次加载）
   - 主题文件（如 `debugging.md`）按需读取
   - 同 git 仓库下所有 worktree 共享

#### Hooks 系统（生命周期钩子）

Claude Code 最强大的可扩展机制。支持 4 种 handler 类型：command（Shell）、http、prompt（LLM 判断）、agent（子代理验证）。

**生命周期事件**：

| 事件 | 触发时机 |
|------|----------|
| `SessionStart/End` | 会话开始/结束 |
| `UserPromptSubmit` | 用户提交 prompt 后 |
| `PreToolUse` | 工具调用前（可拦截） |
| `PostToolUse` | 工具调用成功后 |
| `PostToolUseFailure` | 工具调用失败后 |
| `PermissionRequest` | 权限弹窗出现时 |
| `Notification` | 通知发送时 |
| `SubagentStart/Stop` | 子代理启停 |
| `Stop` | Claude 完成回复 |
| `PreCompact/PostCompact` | 上下文压缩前后 |
| `InstructionsLoaded` | CLAUDE.md 加载时 |
| `ConfigChange` | 配置文件变更时 |
| `WorktreeCreate/Remove` | Worktree 创建/删除时 |

**典型用法**：
- `PreToolUse` + Bash matcher → 拦截危险命令
- `PostToolUse` + Write/Edit matcher → 自动 lint
- `Notification` → 桌面通知（任务完成、需要确认）
- Hooks 安全：启动时快照，运行时修改需 review

#### MCP 集成（Model Context Protocol）

- 支持 stdio / http / sse / ws 四种传输协议
- 三种作用域：local / project / user
- `.mcp.json` 项目级共享配置
- 支持 OAuth 2.0 认证
- **Tool Search**：MCP 工具过多时自动按需加载（超过上下文 10%）
- **Claude Code 自身可作为 MCP Server**：`claude mcp serve`
- 支持 MCP elicitation（服务器向用户请求输入）
- 支持 MCP resources（@-mention 引用）
- 企业管理：`managed-mcp.json` 强制配置 / 白名单黑名单

#### 会话管理

- `--continue` 恢复最近会话
- `--resume` 会话选择器（搜索、预览、重命名、分支过滤）
- `--from-pr <number>` 恢复 PR 关联会话
- 命名会话，按 git 分支分组
- `/rewind` 和 `--fork-session` 分叉会话

#### 并行工作（Git Worktree）

- `--worktree <name>` 创建隔离工作目录
- 子代理也可用 worktree 隔离
- 退出时自动清理（无修改）或提示保留
- 支持非 git 版本控制（通过 WorktreeCreate/Remove hooks）

#### 权限与安全

- 权限模式：default / acceptEdits / dontAsk / bypassPermissions / plan
- Plan Mode：只读分析，不修改代码
- 工具级权限控制（deny 数组）

#### CI/CD 集成

- GitHub Actions / GitLab CI/CD
- GitHub Code Review（自动 PR 审查）
- Slack 集成（bug 报告 → PR）
- Chrome 扩展（调试 Web 应用）

#### Headless 模式

- `claude -p "prompt"` 无交互执行
- 支持管道：`cat file | claude -p "分析" > output`
- 可指定输出格式：`--output-format json/stream-json`

## 2. Kiro

**定位**：Agentic 开发环境，基于 AWS，重新发明开发者与 AI Agent 的协作方式。

### 2.1 核心理念

- Agent 需要**结构化输入**而非仅自然语言
- 需要正确的**界面和交互方法**来与 Agent 大规模协作
- Kiro 是**真正的协作者**，而非只替你干活

### 2.2 Spec-Driven Development（规格驱动开发）

Kiro 首创的开发模式，将 prompt 转化为结构化的：
- **Requirements**（需求文档）
- **Design**（设计文档）
- **Tasks**（可执行任务列表）

然后由 Agent 按任务实现。这与 NaughtyAgent 的 SDD 方式不谋而合。

### 2.3 Agent Hooks（自动化钩子）

在后台自动触发的 Agent 任务：
- 文件保存时自动更新文档
- 代码变更时自动生成单元测试
- 自动优化性能
- 用户可自定义 hook 规则

### 2.4 Powers（能力系统）

可安装的能力包，封装了 MCP + steering（引导） + hooks：
- **一键安装**，自动配置
- **按需激活**：根据对话上下文动态加载
- 覆盖设计→开发→部署→可观测性全链路

**典型 Powers**：
- Figma → 代码（设计转代码）
- Stripe 支付集成
- Terraform 部署
- Supabase 后端
- Postman API 测试
- Datadog / Sentry 监控
- Snyk 安全扫描
- GitHub / GitLab CI/CD

### 2.5 Autonomous Agent（自主代理）

- 在隔离沙箱环境中独立运行
- 任务完成后自动开 PR
- **跨仓库**工作：一次变更影响多个仓库
- **持续学习**：从你的代码库、工单、反馈中学习
- 团队共享：协调多人并行工作

### 2.6 CLI

- Kiro 也提供命令行版本
- 可在终端中使用核心 Agent 功能

## 3. Cursor

**定位**：AI-first 代码编辑器（基于 VS Code fork），用 AI 让你**极致高效**。

### 3.1 核心功能

#### Agent Mode
- 自主完成多文件编辑
- 自动读取代码、搜索上下文
- 执行终端命令
- 支持 MCP 工具调用

#### Tab Completion
- 智能代码补全
- 多行编辑预测
- 基于当前编辑上下文

#### Inline Edit（Cmd+K）
- 选中代码后用自然语言编辑
- 生成代码 / 修改代码 / 解释代码

#### @-mention 上下文
- @file / @folder / @code / @web / @docs / @git
- 手动精确控制 Agent 可见的上下文

#### Rules（规则系统）
- `.cursor/rules/` 目录下的规则文件
- 项目级和用户级规则
- 类似 Claude Code 的 CLAUDE.md

#### Background Agent（后台代理）
- 在云端 VM 中运行
- 可从 GitHub issue 自动开始工作
- 完成后开 PR
- 类似 Kiro 的 Autonomous Agent

#### Bug Finder
- 主动扫描代码中的 bug
- 静态分析 + AI 推理

### 3.2 上下文管理

- 自动索引整个代码库
- 语义搜索（embedding based）
- `.cursorignore` / `.cursorindexingignore` 控制范围
- 文档集成：@docs 引用官方文档

## 4. Windsurf (Cascade)

**定位**：下一代 AI IDE，核心是 **Cascade** 对话式 Agent。

### 4.1 Cascade 核心功能

#### 双模式
- **Code 模式**：可修改代码库
- **Chat 模式**：只回答问题，提出建议

#### 计划系统
- 内置规划 Agent，后台持续优化长期计划
- 自动创建 Todo 列表跟踪复杂任务
- 可在对话中修改计划

#### 实时感知
- **独特能力**：感知用户的实时操作
- 无需重复提供上下文，直接说"继续"

#### 工具调用
- Search / Analyze / Web Search / MCP / Terminal
- 自动检测并安装缺失的包和工具
- Linter 集成：自动修复生成代码的 lint 错误

#### 检查点和回滚
- 命名检查点（Named Checkpoints）
- 可回滚到任意步骤的代码状态
- 类似 git 但更细粒度

#### 并行 Cascade
- 多个 Cascade 同时运行
- 支持 Worktrees 隔离
- 可 @-mention 引用之前的对话

#### 其他
- 语音输入
- 消息队列（等待时预排后续指令）
- 分享对话轨迹
- `.codeiumignore` 文件忽略

### 4.2 记忆系统（Memories）

- 对话间记忆保持
- 自动学习用户偏好和项目模式

## 6. 对 NaughtyAgent 的启示

### 6.1 架构质量评估

NaughtyAgent 当前已有的能力与商业产品对比：

| 能力 | NaughtyAgent 状态 | 商业标杆 | 差距 |
|------|-------------------|----------|------|
| Agent Loop（流式/错误恢复/中断） | ✅ 已有 | Claude Code | 持平 |
| 工具系统 | ✅ 7+6 工具 | Claude Code ~10 | 接近 |
| LLM Provider | ✅ 3个 | Claude Code 主用 Anthropic | 超越 |
| 子代理 | ✅ 已有 | Claude Code 4种+自定义 | 需增强自定义能力 |
| MCP Client | ✅ 已有 | 全员支持 | 持平 |
| 安全/权限 | ✅ 3级权限 | Claude Code 5种模式 | 可增强 |
| 记忆系统 | ❌ 未实现 | Claude Code 双层记忆 | **重点差距** |
| Hooks 生命周期 | ❌ 未实现 | Claude Code 20+事件 | **重点差距** |
| Spec 驱动开发 | ✅ SDD 已有 | Kiro 首创 | 持平/可学习 |
| 会话管理 | ✅ 基础 | Claude Code 完善 | 需增强 |
| Worktree 隔离 | ❌ 未实现 | Claude Code + Windsurf | 可后期做 |
| Headless 模式 | ❌ 未实现 | Claude Code | 可后期做 |

### 6.2 最值得借鉴的功能（按优先级）

1. **记忆系统**（Claude Code 双层记忆）
   - CLAUDE.md 等价物已有（项目指令文件）
   - 缺少 Auto Memory（Agent 自主积累知识）
   - 影响：跨会话体验的关键

2. **Hooks/生命周期系统**（Claude Code）
   - 可扩展性的核心
   - PreToolUse → 安全拦截
   - PostToolUse → 自动 lint/格式化
   - 建议按需实现，不必做全部 20+事件

3. **Powers/Plugin 系统**（Kiro）
   - 能力打包 + 一键安装
   - 对标 Claude Code 的 Plugins + Skills
   - 长期竞争力的关键

4. **Plan Mode**（Claude Code）
   - 只读分析模式，安全探索代码库
   - 实现成本低，用户体验好

5. **检查点/回滚**（Windsurf）
   - 细粒度代码状态管理
   - 用户安全感的保障

### 6.3 不必急于实现的功能

- Tab 补全（需要 IDE 深度集成，VS Code 已有生态）
- 实时感知（Windsurf 独有，实现复杂）
- 后台云端 Agent（需要服务器基础设施）
- 企业管理策略（面向个人使用暂不需要）

### 6.4 架构建议

你的架构选择是**正确的**：
- **分离式架构**（Agent 服务 + VS Code UI）与 Claude Code 一致
- **直接调 API**而非套壳 Agent SDK 是对的
- **SDD 开发方式**与 Kiro 理念一致

**建议的演进路线**：
```
当前（72%）→ 补记忆系统 → 加 Hooks → Plan Mode → Plugin/Powers → 检查点
```
