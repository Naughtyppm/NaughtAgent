# 自我进化系统 - 最终方案

> 融合事件驱动 + 日志追溯 + 权限控制的完整自我改进架构

---

## 一、核心架构

```
LLM 执行任务
  ↓ 多次尝试后成功
  ↓ 自我审查
  ↓ 发出事件 <event:skill.improvement.identified>
  ↓
experience-distiller 触发
  ↓
  ├─ 1. 搜索现有记录（去重）
  │   └─ grep "Pattern-Key: xxx" .learnings/
  │
  ├─ 2. 记录到 .learnings/LEARNINGS.md
  │   ├─ 生成 ID: LRN-20260314-001
  │   ├─ Pattern-Key: 去重标识
  │   └─ Recurrence-Count: 重复次数
  │
  ├─ 3. 判断是否创建/更新 Skill
  │   ├─ Recurrence-Count < 3 → 只记录
  │   ├─ Recurrence-Count >= 3 → 检查权限
  │   └─ 有权限 → 创建/更新 Skill
  │
  └─ 4. 检查权限配置
      ├─ autoFix.enabled = true → 自动修复
      └─ autoFix.enabled = false → 只记录
  ↓
下次遇到类似情况
  ↓ 事件触发
  ↓ Skill 注入提示词
  ↓ 避免重复失败
```

## 三、核心组件

### 3.1 系统级 Steering

**文件：** `.kiro/steering/llm-events.md`

**功能：**
- 定义事件格式
- 自我审查机制
- 触发条件

### 3.2 experience-distiller

**功能：**
1. 搜索现有记录（去重）
2. 记录到 .learnings/
3. 更新 Recurrence-Count
4. 判断是否创建 Skill
5. 检查权限配置

**关键逻辑：**
```
Recurrence-Count = 1 → 只记录
Recurrence-Count = 2 → 更新记录
Recurrence-Count >= 3 → 创建/更新 Skill
```

### 3.3 skill-creator

**功能：**
- 创建新 Skill 文件
- 定义事件订阅
- 关联日志记录


### 3.4 权限配置

**文件：** `.naughtyagent/permissions.json`

**功能：**
- 定义可修改的代码范围
- 设置修复条件
- 三级权限控制

## 四、完整示例

### 场景：firecrawl 工具参数错误

**第 1 次遇到：**
```
调用 firecrawl_scrape，formats: "markdown"
  ↓ 报错
  ↓ 改成 formats: ["markdown"]
  ↓ 成功
  ↓ 发出事件

experience-distiller:
  ├─ 搜索：无记录
  ├─ 创建：LRN-20260314-001
  ├─ Pattern-Key: mcp.firecrawl.formats
  └─ Recurrence-Count: 1 → 只记录
```


**第 2 次遇到：**
```
一周后，又犯同样错误
  ↓ 发出事件

experience-distiller:
  ├─ 搜索：找到 LRN-20260314-001
  ├─ 更新：Recurrence-Count: 2
  └─ Last-Seen: 2026-03-21 → 继续观察
```

**第 3 次遇到：**
```
又犯同样错误
  ↓ 发出事件

experience-distiller:
  ├─ 搜索：找到 LRN-20260314-001
  ├─ 更新：Recurrence-Count: 3 → 达到阈值！
  ├─ 检查权限：Skills 可以创建
  └─ 调用 skill-creator

skill-creator:
  └─ 创建 Skills/firecrawl-usage/SKILL.md
```

**下次自动应用：**
```
再次调用 firecrawl_scrape
  ↓ 触发事件
  ↓ firecrawl-usage Skill 注入提示词
  ↓ 直接使用正确格式
  ↓ 一次成功！
```

## 五、关键特性

### 5.1 去重机制
- Pattern-Key 标识
- 搜索现有记录
- 避免重复创建 Skill

### 5.2 阈值控制
- Recurrence-Count >= 3 才创建
- 避免为一次性问题创建 Skill

### 5.3 权限控制
- 用户配置可修改范围
- 三级权限（自动/建议/只记录）

### 5.4 双重保障
- 日志系统：完整追溯
- Skill 系统：自动应用
