# NaughtyAgent 产品概览

> 一个类型安全、事件驱动、具备智能错误恢复能力的 AI Agent 框架

---

## 一、核心特点

### 1.1 三大核心优势

**1. 类型安全的工具系统**
- 使用 Zod Schema 定义工具参数，编译时和运行时双重保证
- 自动生成 JSON Schema 给 LLM，减少调用错误
- 统一的 Tool.define() 模式，易于扩展

**2. 智能错误恢复机制（业界领先）**
- 自动检测重复错误，防止死循环
- 主动提供恢复策略（如：截断错误 → 建议分段写入）
- 提升 Agent 自主性，减少人工干预

**3. 流式输出 + 事件驱动架构**
- 实时看到 AI 思考过程，用户体验优秀
- UI 层完全解耦，支持多种界面（CLI、VS Code、Web）
- 支持 Extended Thinking 模式

### 1.2 架构设计亮点

**Loop 控制权在 LLM**
- 代码提供框架（while + maxSteps），LLM 控制流程
- 自动分解多步骤任务，无需预知步数
- 观察-决策循环，实现真正的"智能体"

**工具系统可扩展**
- 内置 6 大核心工具（read/write/edit/bash/glob/grep）
- 支持 MCP（Model Context Protocol）集成外部工具
- 3 步添加新工具（定义 → 注册 → 使用）

**Skills 系统兼容**
- 100% 兼容 Claude Code/Kiro Skills 格式
- 支持事件驱动的 Skill 激活
- 可共享社区 Skills

## 三、核心能力

### 3.1 内置工具（6 个）

| 工具 | 功能 | 特点 |
|------|------|------|
| read | 读取文件 | 支持行范围，减少 token |
| write | 写入文件 | 自动创建目录 |
| edit | 编辑文件 | 文本替换（待升级为 AST） |
| bash | 执行命令 | 超时控制，输出截断 |
| glob | 搜索文件 | 使用 glob 模式 |
| grep | 搜索内容 | 正则搜索，显示上下文 |

### 3.2 智能特性

**错误恢复策略：**
- 文件不存在 → 建议使用 glob 确认路径
- 内容截断 → 建议使用 write + append 分段写入
- edit 失败 → 建议检查空格和换行
- 重复错误 → 自动切换策略

**上下文管理：**
- 输出自动截断（10000 字符）
- 提供分段读取建议
- 支持行范围精确读取

### 3.3 扩展能力

**MCP 工具集成：**
- 支持外部工具服务器
- 统一的工具调用接口
- 社区工具生态

**Skills 系统：**
- 加载自定义 Skills
- 事件驱动激活
- 会话标识显示

## 八、快速开始

### 8.1 基础使用

```typescript
import { Agent } from 'naughty-agent'

// 创建 Agent
const agent = new Agent({
  provider: 'anthropic',
  model: 'claude-3-5-sonnet',
  tools: ['read', 'write', 'bash'],
  maxSteps: 100
})

// 执行任务
for await (const event of agent.run("帮我重构这个函数")) {
  if (event.type === 'text') {
    console.log(event.content)
  }
}
```

### 8.2 添加自定义工具

```typescript
import { Tool } from 'naughty-agent'

// 定义工具
const myTool = Tool.define({
  id: "my_tool",
  description: "我的自定义工具",
  parameters: z.object({
    input: z.string()
  }),
  execute: async (params, ctx) => {
    // 实现逻辑
    return { output: "结果" }
  }
})

// 注册工具
agent.registerTool(myTool)
```

### 8.3 加载 Skills

```typescript
// 加载 Skill
agent.loadSkill('experience-distiller')

// 激活 Skill
agent.activateSkill('experience-distiller', {
  trigger: 'manual'
})
```
