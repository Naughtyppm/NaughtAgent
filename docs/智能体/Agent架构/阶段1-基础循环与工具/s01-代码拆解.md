# s01 Agent Loop - 代码拆解对比

## 教材核心代码（Python）

```python
def agent_loop(messages: list):
    while True:
        # 1. 调用 LLM
        response = client.messages.create(
            model=MODEL, 
            system=SYSTEM,
            messages=messages,
            tools=TOOLS,
            max_tokens=8000,
        )
        
        # 2. 保存 LLM 回复
        messages.append({"role": "assistant", "content": response.content})
        
        # 3. 检查是否需要调用工具
        if response.stop_reason != "tool_use":
            return  # 不需要工具，结束
        
        # 4. 执行工具
        results = []
        for block in response.content:
            if block.type == "tool_use":
                output = run_bash(block.input["command"])
                results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": output
                })
        
        # 5. 把工具结果喂回去，继续循环
        messages.append({"role": "user", "content": results})
```

**核心逻辑**：5 个步骤的循环，模型控制何时停止。

## NaughtyAgent 核心代码（TypeScript）

从 `loop.ts` 的 `executeLoop` 函数提取核心：


```typescript
async function* executeLoop(input: string): AsyncGenerator<AgentEvent> {
  // 添加用户消息
  addMessage(session, "user", [{ type: "text", text: input }])
  
  let stepCount = 0
  const maxSteps = 100  // 安全阀：最多 100 步
  
  while (stepCount < maxSteps) {
    stepCount++
    
    // 1. 调用 LLM（流式）
    const messages = convertMessages()  // 转换 session 消息格式
    let responseText = ""
    const toolCalls = []
    
    for await (const event of provider.stream({
      model: modelConfig,
      messages,
      system: systemPrompt,
      tools: tools,
    })) {
      if (event.type === 'text') {
        responseText += event.text
        yield { type: "text", content: responseText }  // 实时输出
      } else if (event.type === 'tool_call') {
        toolCalls.push(event)
      }
    }
    
    // 2. 保存 LLM 回复
    const assistantContent = []
    if (responseText) {
      assistantContent.push({ type: "text", text: responseText })
    }
    for (const toolCall of toolCalls) {
      assistantContent.push({
        type: "tool_use",
        id: toolCall.id,
        name: toolCall.name,
        input: toolCall.args
      })
    }
    addMessage(session, "assistant", assistantContent)
    
    // 3. 检查是否需要调用工具
    if (toolCalls.length === 0) {
      break  // 不需要工具，结束
    }
    
    // 4. 执行工具
    const toolResults = []
    for (const toolCall of toolCalls) {
      const { result, isError } = await executeTool(toolCall)
      toolResults.push({
        type: "tool_result",
        tool_use_id: toolCall.id,
        content: result.output,
        is_error: isError
      })
    }
    
    // 5. 把工具结果喂回去，继续循环
    addMessage(session, "user", toolResults)
  }
  
  yield { type: "done", usage: totalUsage }
}
```

**与教材的对应关系**：

| 步骤 | 教材 | NaughtyAgent |
|------|------|-------------|
| 循环条件 | `while True` | `while (stepCount < maxSteps)` |
| 调用 LLM | `client.messages.create()` | `provider.stream()` |
| 保存回复 | `messages.append()` | `addMessage(session, "assistant", ...)` |
| 停止判断 | `if stop_reason != "tool_use"` | `if (toolCalls.length === 0)` |
| 执行工具 | `run_bash()` | `executeTool()` |
| 喂回结果 | `messages.append()` | `addMessage(session, "user", ...)` |

## 关键差异

### 1. 循环控制

**教材**：
```python
while True:  # 无限循环
    if response.stop_reason != "tool_use":
        return  # 模型决定停止
```

**NaughtyAgent**：
```typescript
while (stepCount < maxSteps) {  // 最多 100 步
    if (toolCalls.length === 0) {
        break  // 模型决定停止
    }
}
```

差异：NaughtyAgent 加了 `maxSteps` 安全阀，防止无限循环。


### 2. 流式输出

**教材**：
```python
response = client.messages.create(...)  # 同步调用，一次性返回
print(output[:200])  # 执行完才打印
```

**NaughtyAgent**：
```typescript
for await (const event of provider.stream(...)) {  // 流式调用
  if (event.type === 'text') {
    yield { type: "text", content: responseText }  // 实时输出
  }
}
```

差异：NaughtyAgent 用流式 API，用户能实时看到 LLM 的思考过程。

### 3. 消息存储

**教材**：
```python
messages = []  # 就是个数组
messages.append({"role": "assistant", "content": ...})
```

**NaughtyAgent**：
```typescript
addMessage(session, "assistant", ...)  // 写入 Session 对象
```

差异：NaughtyAgent 把消息持久化到 Session，教材只在内存里。

## 本质相同的部分

虽然实现细节不同，但**核心模式完全一致**：

```
1. 调用 LLM
2. 保存回复
3. 如果有 tool_use → 执行工具 → 喂回结果 → 回到步骤 1
4. 如果没有 tool_use → 结束
```

这就是 Agent Loop 的本质。教材 80 行，NaughtyAgent 650 行，但循环逻辑是同构的。

## 延伸思考：maxSteps 安全阀的改进

### 问题

固定 `maxSteps = 100` 太死板。如果上下文压缩做好了，任务确实需要 200 步怎么办？

### 安全阀不能去掉

防的不是正常执行，是模型陷入死循环（反复修改 → 测试失败 → 再修改）。

### 更好的方案：动态安全阀

结合 s03 的 TodoManager，判断"是否在推进"而非"跑了多少步"：

```
如果 todo 列表在推进（有任务从 pending → in_progress → completed）
  → 允许继续
如果连续 N 步没有 todo 进展
  → 提前终止，提示模型反思
```

这是 s01（循环）和 s03（规划）的交叉点。
