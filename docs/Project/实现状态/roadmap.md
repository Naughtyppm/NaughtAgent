## 开发原则

1. **小步快跑** - 每个版本聚焦 2-3 周可完成的目标
2. **测试先行** - 新功能必须有测试覆盖
3. **向后兼容** - API 变更需要迁移路径
4. **文档同步** - 功能完成同时更新文档
