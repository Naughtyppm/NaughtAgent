# NaughtAgent VSCode UI 设计建议

> 版本：v1.0 | 日期：2026-04-04 | 基于现有实现分析

## 1. 整体设计原则
### 1.1 核心理念
- **渐进披露**：默认隐藏复杂信息，按需展开
- **状态清晰**：用户始终知道 Agent 在做什么
- **快速响应**：优先显示文本流，工具状态异步更新
- **VSCode 原生**：遵循 VSCode 的设计语言和交互习惯

### 1.2 设计约束
- 基于 WebView 实现，需要自己实现 UI 组件
- 只能使用 HTML/CSS/JS，无法使用 React/Vue 等框架
- 需要与 VSCode 主题保持一致
- 移动端适配不是必需品，专注桌面体验

## 2. Chat 界面设计建议

### 2.1 整体布局（从上到下）
```
┌─────────────────────────────────────────────────────┐
│ [Header] Model: Claude | Session: main | ●connected │
├─────────────────────────────────────────────────────┤
│ [Messages Area - Scrollable]                       │
│ ┌─ User ─────────────────────────────────────────┐  │
│ │ 帮我分析这个函数的性能问题                       │  │
│ └─────────────────────────────────────────────────┘ │
│ ┌─ Assistant - Streaming ───────────────────────┐  │
│ │ 我来分析... [Thinking] [Tool] read ✓           │  │
│ └─────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────┤
│ [Input] 输入消息... [Send/Stop]                    │
└─────────────────────────────────────────────────────┘
```

### 2.2 核心 UI 组件设计

**Header Bar**：
- 左：模型选择 (Claude Sonnet ▼)
- 中：会话名称 (Session: main ▼)  
- 右：连接状态 (● connected)

**消息卡片**：
- User: 蓝色左边框，浅色背景
- Assistant: 默认背景，带边框
- Thinking: 虚线边框，斜体，可折叠

**工具状态**：
- [Tool] read file.ts ✓ (成功)
- [Tool] edit file.ts ⏳ (执行中)
- [Tool] bash test ❌ (失败)

**权限确认**：Modal 弹窗，显示操作详情，[预览][允许][拒绝]

### 2.3 实现要点

1. **流式显示**：文本逐步显示，带光标动画
2. **工具折叠**：长输出默认折叠，显示前几行
3. **主题适配**：使用 VSCode CSS 变量
4. **性能优化**：虚拟滚动，输出截断
