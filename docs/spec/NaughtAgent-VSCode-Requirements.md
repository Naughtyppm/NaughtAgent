# NaughtAgent VSCode 集成 — 需求文档

> 版本：v1.0 | 日期：2026-04-04 | 状态：初稿

## 1. 需求背景

NaughtAgent 是类 Claude Code 的 AI 编程助手，当前以 CLI（终端）为主要交互界面。项目已有：
- **CLI plain-text 模式**（v0.9.3+）：流式输出、折叠、虚拟滚动、权限对话
- **CLI ink 模式**：React Terminal UI，功能完整但性能不稳定
- **VSCode 扩展 v0.1.0**：骨架已搭建（Daemon 连接、会话管理、命令注册），但 **Chat Webview 是空壳**
- **Daemon 后端**：HTTP + WebSocket 双通道，已可在 localhost:31415 运行

**核心目标**：让 NaughtAgent 在 VSCode 中提供完整的 AI 编程体验，对标 GitHub Copilot Chat / Cursor / Claude Code VSCode 集成。

---

## 2. 功能需求

### FR-1: Chat Webview 核心界面（P0）

**描述**：在 VSCode 侧边栏提供完整的 Chat 对话界面。

**验收标准**：
- AC-1.1: 用户可在侧边栏打开 Chat 面板
- AC-1.2: 支持文本输入、发送消息、接收流式响应
- AC-1.3: AI 回复支持 Markdown 渲染（代码块语法高亮）
- AC-1.4: 工具调用过程可视化（折叠/展开，显示工具名+输入+输出）
- AC-1.5: 思考过程（thinking）可折叠显示
- AC-1.6: 支持 `/` 命令（/help、/clear、/model、/thinking）

### FR-2: 流式消息处理（P0）

**描述**：实时流式显示 AI 生成的文本、工具调用、思考过程。

**验收标准**：
- AC-2.1: 文本逐字流式显示，无延迟累积
- AC-2.2: 工具调用开始时显示 spinner + 工具名
- AC-2.3: 工具调用结束显示结果摘要（可折叠）
- AC-2.4: Token 使用统计实时更新
- AC-2.5: 支持取消正在进行的请求

### FR-3: 权限确认界面（P0）

**描述**：工具操作需用户确认时，在 Webview 中显示确认对话框。

**验收标准**：
- AC-3.1: 文件写入/编辑/删除请求弹出确认对话框
- AC-3.2: Bash 命令执行弹出确认（显示完整命令）
- AC-3.3: 支持"允许本次"/"始终允许此类操作"/"拒绝"三选项
- AC-3.4: 自动确认配置生效（已在 settings 中定义）

### FR-4: 文件变更预览与应用（P1）

**描述**：AI 修改文件时，提供 Diff 预览和逐步应用。

**验收标准**：
- AC-4.1: 文件变更以内联 Diff 展示（绿色增/红色删）
- AC-4.2: 点击文件名可在编辑器中打开对应 Diff 视图
- AC-4.3: 支持"应用全部"/"逐项应用"/"拒绝"
- AC-4.4: 应用后自动保存文件

### FR-5: 上下文感知（P1）

**描述**：自动收集编辑器上下文，增强 AI 回复质量。

**验收标准**：
- AC-5.1: 自动附加当前打开的文件信息
- AC-5.2: 选中代码自动作为上下文
- AC-5.3: 支持 `@file <path>` 语法引用文件
- AC-5.4: 支持 `@workspace` 引用整个工作区结构
- AC-5.5: @file 支持路径补全

### FR-6: 会话管理（P1）

**描述**：支持多会话切换、历史对话恢复。

**验收标准**：
- AC-6.1: 新建/切换/删除会话
- AC-6.2: 会话历史持久化（跨 VSCode 重启）
- AC-6.3: 会话列表显示时间、消息数、Agent 类型

### FR-7: CLI 完善 — 记忆加载改进（P1）

**描述**：CLI 启动时更智能地加载和展示持久记忆。

**验收标准**：
- AC-7.1: 启动时自动检测并提示 `.naughty/memory.md` 存在
- AC-7.2: REPL 中新增 `/memory` 命令查看/编辑记忆
- AC-7.3: 会话结束时提示保存有价值的信息到记忆
- AC-7.4: 记忆内容在系统提示词中的注入保持当前行为

### FR-8: 状态栏集成（P2）

**描述**：在 VSCode 状态栏显示 NaughtAgent 状态。

**验收标准**：
- AC-8.1: 显示 Daemon 连接状态（图标+文字）
- AC-8.2: 显示当前会话的 Agent 类型和模型
- AC-8.3: 点击可快速切换模型/Agent
- AC-8.4: 显示当前会话 Token 消耗

## 3. 非功能需求

### NFR-1: 性能
- Webview 渲染延迟 < 100ms
- 流式文本输出延迟 < 50ms
- Daemon 健康检查间隔 30s，不占用显著 CPU

### NFR-2: 可靠性
- Daemon 断线自动重连（指数退避，max 30s）
- WebSocket 异常断开时保留当前对话，重连后继续

### NFR-3: 兼容性
- VSCode >= 1.85.0
- 支持 Windows / macOS / Linux
- 支持 Remote（SSH、WSL、Codespaces）

### NFR-4: 安全性
- WebSocket 仅绑定 localhost
- 权限系统不可绕过
- 文件操作在沙箱路径内

## 4. 用户故事

### US-1: 基础对话
> 作为开发者，我想在 VSCode 侧边栏与 NaughtAgent 对话，让它帮我读代码、写代码、调试问题，不需要切换到终端。

### US-2: 代码辅助
> 作为开发者，我选中一段代码后右键"询问 NaugtyAgent"，AI 自动获取选中代码和文件上下文，给出分析/建议/修复方案。

### US-3: 全自动编程
> 作为开发者，我描述需求后，NaughtAgent 自主读取项目文件、修改代码、运行验证，每一步变更我都能实时看到 Diff 并决定是否应用。

### US-4: 跨会话记忆
> 作为长期用户，NaughtAgent 记住我的项目偏好、架构决策、已解决的问题，下次对话时不需要重新解释。

---

## 5. 范围与排除

### 在范围内
- VSCode Chat Webview 完整实现
- 流式消息 + 工具可视化
- 权限确认 UI
- Diff 预览与应用
- CLI `/memory` 命令完善
- 上下文收集增强

### 不在范围内（未来版本）
- 内联代码补全（类 Copilot 的 ghost text）
- 多文件批量编辑 UI
- 远程 Daemon 支持
- 语音输入
- 团队协作功能（Agent Teams UI）
