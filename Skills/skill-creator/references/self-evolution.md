# 自进性框架（Self-Evolution Framework）

## 概述

自进性是指 Skill 具备自我纠错、自我更新的能力。当执行过程中发现问题时，能够：
1. 识别问题根因
2. 尝试自动修复
3. 更新文档以防止复发
4. 记录学习经验

## 核心机制

### 1. 错误检测与自动修复

```
执行 Skill 操作
    │
    ├── 成功 → 继续
    │
    └── 失败 → 进入自修复流程
              │
              ├── 分析错误类型
              │   ├── API 参数错误 → 查阅文档，修正参数
              │   ├── 路径/格式错误 → 自动修正
              │   ├── 依赖缺失 → 提示安装
              │   └── 未知错误 → 记录并报告
              │
              ├── 尝试修复（最多 3 次）
              │
              └── 修复成功 → 触发文档更新检查
```

### 2. 文档自更新触发条件

| 触发条件 | 更新内容 | 示例 |
|----------|----------|------|
| API 参数与文档不符 | 修正参数说明 | `defer_compile` → `deferCompile` |
| 发现更优实践 | 添加最佳实践 | "批量操作时先 defer" |
| 常见错误模式 | 添加 FAQ/陷阱 | "路径必须带 /Game/ 前缀" |
| 新功能/变更 | 更新功能列表 | 新增 API 端点 |

### 3. 自更新流程

```markdown
## 文档更新流程

1. **检测到不一致**
   - 执行失败且原因是文档描述与实际不符
   - 或发现文档缺失重要信息

2. **询问用户确认**（可选，取决于 Skill 配置）
   ```
   发现文档与实际 API 不符：
   - 文档描述：参数名为 `defer_compile`
   - 实际行为：参数名为 `deferCompile`

   是否更新 Skill 文档？[Y/n]
   ```

3. **执行更新**
   - 使用 Edit 工具修改对应的 .md 文件
   - 在更新日志中记录变更

4. **验证更新**
   - 重新执行操作确认修复有效
```

### 4. 学习日志规范

每个 Skill 应维护一个学习日志文件：

```
{skill-name}/
├── SKILL.md
├── references/
└── logs/
    └── learnings.jsonl    # 学习记录（JSON Lines 格式）
```

**日志格式**：

```json
{
  "timestamp": "2026-02-02T10:30:00Z",
  "type": "error_fix|doc_update|best_practice",
  "trigger": "执行 /compile_blueprint 失败",
  "problem": "参数 defer_compile 无效",
  "solution": "参数名应为 deferCompile（驼峰命名）",
  "doc_updated": true,
  "file_changed": "L2-simple-member.md:45"
}
```

## 在 SKILL.md 中添加自进性章节

每个 Skill 应包含以下章节：

```markdown
---

## 自进性机制

### 错误自修复

当执行失败时：
1. 分析错误信息，识别根因
2. 查阅本文档的「常见问题」章节
3. 尝试自动修复（最多 3 次不同方案）
4. 修复成功后检查是否需要更新文档

### 文档自更新

发现以下情况时，应更新本 Skill 文档：
- API 参数/格式与文档描述不符
- 发现文档未记录的重要行为
- 积累了可复用的最佳实践

**更新流程**：
1. 询问用户："发现 {问题}，是否更新 Skill 文档？"
2. 用户同意后，使用 Edit 工具修改对应文件
3. 在 `logs/learnings.jsonl` 中记录变更

### 常见问题（FAQ）

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| {问题1} | {原因} | {方案} |
| {问题2} | {原因} | {方案} |

*此表格会随着使用经验自动扩充*
```

## 自进性等级

| 等级 | 描述 | 适用场景 |
|------|------|----------|
| L0 - 无 | 不具备自进性 | 简单工具类 Skill |
| L1 - 被动 | 仅记录错误，不自动修复 | 低风险操作 |
| L2 - 主动 | 自动尝试修复，询问后更新文档 | 大多数 Skill |
| L3 - 自主 | 自动修复 + 自动更新文档 | 高频使用的成熟 Skill |

## 实现检查清单

为 Skill 添加自进性时，确保：

- [ ] 添加「自进性机制」章节
- [ ] 定义错误处理流程
- [ ] 创建 FAQ 表格（初始可为空）
- [ ] 指定自进性等级
- [ ] 创建 `logs/` 目录（可选）
- [ ] 在 description 中提及自更新能力（可选）
