# 🎯 你的最终路线（一句话）

> **以 Zed 为交互范式
>  用 Neovim / VS 做纯编辑
>  用一个“去人格化的开源 Agent 底座”当核心
>  所有智能显式触发、IDE 永远无感**

------

# 🧭 Step 0：你现在所处的位置（先确认）

你已经具备：

- ✅ 裸奔 IDE 的判断力
- ✅ 不信任 IDE 内置 agent
- ✅ 已经在仿 Zed 的 AI 交互模型
- ✅ 想长期维护，而不是 demo

说明你不是在选工具，而是在**定体系**。

------

# 🧱 Step 1：确定“唯一参照物”

**把 Zed 当成“交互标准”，不是产品**

你只继承它三件事：

1. **AI 永不自动运行**
2. **一切从选区 / 明确指令开始**
3. **Apply / Reject 必须在人手里**

你不追：

- 它的 UI
- 它的 Rust 实现
- 它的协作功能

👉 **你要的是“Zed 的态度”，不是 Zed 本身**

------

# 🧩 Step 2：固定你的编辑器层（一次性定死）

选一个，不要反复横跳：

### 你的主编辑器

- **Neovim（已有）**
- 或 **VS Code（阉割智能）**

原则只有一条：

> 编辑器 **永远不知道 agent 存在**

你之后的所有设计都不能破坏这一点。

------

# 🤖 Step 3：选定 Agent 底座（最终定案）

### ✅ **OpenAI Agents SDK（开源）**

你对它的使用规则是：

- ❌ 不用自动 planning
- ❌ 不用 autonomous loop
- ❌ 不用多 agent 自嗨

你只拿：

- agent / tool 协议
- 调用抽象
- 状态容器

然后：

> **Loop、策略、失败处理，全部你自己写**

这一步定下来，**三年内不用再换**。

------

# 🔌 Step 4：定义“唯一触发方式”（非常关键）

你只允许 agent 被这三种方式叫醒之一：

1. **编辑器插件触发**
   - 选区
   - 明确指令
2. **CLI**
3. **MCP / HTTP**

禁止：

- 后台监听
- 文件变动自动触发
- 定时运行

👉 agent 必须像“工具”，不是“守护进程”。

------

# 🧠 Step 5：你的 AI 插件该长什么样（对齐 Zed）

无论是 Neovim 插件还是 VS 插件，只做四件事：

1. 收集上下文（选区 / 文件 / diff）
2. 展示 prompt（可编辑）
3. 发请求
4. 展示结果 + 让人确认

**插件不允许：**

- 拆任务
- 多轮规划
- 自动改文件

它只是 **Zed 的“对话层复刻”**。

------

# 🗂️ Step 6：项目结构（长期维护友好）

你最终会形成这个结构：

```
editor/
  └─ nvim-plugin / vscode-extension

agent/
  ├─ core/        ← 你的 loop / state / strategy
  ├─ tools/       ← 显式工具
  ├─ prompts/     ← 可版本化
  └─ runtime/     ← CLI / MCP / HTTP

docs/
  └─ philosophy.md  ← 写清楚：为什么不自动
```

📌 重点是：
 **editor 和 agent 永远是两个 repo / 两个世界**

------

# 🧠 Step 7：你未来做判断时的“唯一准绳”

以后你每加一个功能，只问自己一句：

> **如果没有我点按钮，它会不会自己动？**

- 如果答案是“会” → 删掉
- 如果答案是“不会” → 合格

------

# ✅ 最终方案总结（你可以截图）

- 交互范式：**Zed**
- 编辑器：**Neovim / VS（裸）**
- AI 插件：**CodeCompanion / Continue 风格**

- 执行方式：**显式触发**
- 核心原则：**人永远在方向盘上**
