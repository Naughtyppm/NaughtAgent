# Bug：tool_use / tool_result 消息不匹配导致 400

## 现象

Na 执行 TASK-002 时报错：

```
400 {"error":{"message":"{\"message\":\"messages.12: `tool_use` ids were found without `tool_result` blocks
immediately after: toolu_bdrk_01S5nZPwX12biGiWKyJyKArx, toolu_bdrk_01TaJjvRP23zwPESQo7uhx6k. Each `tool_use` block
must have a corresponding `tool_result` block in the next message.\"}","type":"error"}}
```

## 分析

Claude API 要求每个 `tool_use` 块必须在紧接着的下一条消息中有对应的 `tool_result` 块。

可能原因：
1. **compact 截断了 tool_result**：`microCompact()` 或 `autoCompact()` 在压缩消息时可能丢弃了 `tool_result`
2. **错误恢复跳过了 tool_result**：工具执行出错时，可能没有正确插入 `tool_result` 就继续了下一轮
3. **消息构建逻辑缺陷**：`message-converter.ts` 在转换消息时可能丢失了部分 `tool_result`

## 相关文件

- `packages/agent/src/agent/loop.ts` — 工具执行后插入 tool_result
- `packages/agent/src/agent/compact.ts` — 消息压缩（可能丢弃 tool_result）
- `packages/agent/src/agent/message-converter.ts` — 消息格式转换

## 排查步骤

1. 读取 `loop.ts` 中工具执行后的消息插入逻辑，确认每个 tool_use 都有对应 tool_result
2. 读取 `compact.ts` 的 `microCompact()`，确认它不会破坏 tool_use/tool_result 配对
3. 读取 `message-converter.ts`，确认转换不会丢失 tool_result
4. 如有可能，复现场景（多次工具调用 + compact 触发）

## 优先级

P1 — 影响 Na 正常工作
