# MVP Sprint - 需求文档

## 目标

**在 1 周内让 NaughtAgent 能跑起来**，实现最小可用的 AI 编程助手。

## MVP 定义

用户能通过 CLI 与 Agent 对话，Agent 能：
1. 理解用户意图
2. 调用工具（读文件、写文件、执行命令）
3. 返回结果

## 核心场景

### 场景 1：简单对话
```
用户: 你好
Agent: 你好！我是 NaughtAgent，一个 AI 编程助手。有什么可以帮你的？
```

### 场景 2：读取文件
```
用户: 读一下 package.json
Agent: [调用 Read 工具]
       这是 package.json 的内容：
       - name: naughty-agent
       - version: 0.1.0
       ...
```

### 场景 3：修改文件
```
用户: 把 version 改成 0.2.0
Agent: [调用 Edit 工具]
       已将 version 从 0.1.0 修改为 0.2.0
```

### 场景 4：执行命令
```
用户: 运行 npm test
Agent: [调用 Bash 工具]
       测试结果：
       - 152 个测试通过
       - 0 个失败
```

## 功能需求

### MVP-1: Agent Loop（核心）
- [ ] 接收用户消息
- [ ] 调用 LLM（Claude API）
- [ ] 解析 LLM 响应（文本 / 工具调用）
- [ ] 执行工具调用
- [ ] 将工具结果返回给 LLM
- [ ] 循环直到 LLM 返回最终响应
- [ ] 输出最终响应给用户

### MVP-2: 工具执行
- [ ] Read 工具：读取文件内容
- [ ] Write 工具：写入文件内容
- [ ] Edit 工具：编辑文件内容
- [ ] Bash 工具：执行 shell 命令
- [ ] Glob 工具：文件匹配
- [ ] Grep 工具：文本搜索

### MVP-3: CLI 接口
- [ ] 启动交互式会话
- [ ] 接收用户输入
- [ ] 显示 Agent 响应
- [ ] 显示工具调用过程
- [ ] 支持退出命令

## 非功能需求

### MVP-N1: 可观测性
- [ ] 显示当前正在执行的工具
- [ ] 显示 LLM 调用耗时
- [ ] 显示 token 使用量

### MVP-N2: 错误处理
- [ ] LLM 调用失败时重试
- [ ] 工具执行失败时返回错误信息给 LLM
- [ ] 网络超时处理

## 不在 MVP 范围

以下功能**暂不实现**，后续迭代：
- MCP 集成
- 会话分支
- 成本追踪
- VS Code 扩展
- 权限控制
- 多模态（图片）
- 流式输出

## 验收标准

1. `pnpm run cli` 能启动交互式会话
2. 能完成上述 4 个核心场景
3. 工具调用过程可见
4. 错误有友好提示
