# MVP Sprint - 任务清单

## 概述

本任务清单聚焦 MVP 实现，目标是**让 Agent 先跑起来**。

**原则**：
- 每个任务都有明确的验收标准
- 优先跑通流程，再优化细节
- 复用已有代码，不重复造轮子

## 任务列表

### Sprint 1: Agent Loop 核心（Day 1-2）

- [ ] **1.1 实现 LLM Provider**
  - 文件：`packages/agent/src/provider/anthropic.ts`
  - 封装 Anthropic SDK 调用
  - 支持工具定义传递
  - 支持流式响应（可选，MVP 可不做）
  - **验收**：能调用 Claude API 并返回响应

- [ ] **1.2 实现 Agent Loop 核心逻辑**
  - 文件：`packages/agent/src/agent/loop.ts`
  - 实现消息构建
  - 实现 LLM 调用
  - 实现响应解析（文本 vs 工具调用）
  - 实现工具执行
  - 实现循环控制（最大迭代次数）
  - **验收**：能完成一轮对话（无工具调用）

- [ ] **1.3 集成工具调用**
  - 在 Agent Loop 中集成 ToolRegistry
  - 将内置工具转换为 LLM 工具格式
  - 执行工具并将结果返回给 LLM
  - **验收**：能完成带工具调用的对话

- [ ] **1.4 Checkpoint 1 - 核心验证**
  - 编写 Agent Loop 单元测试
  - 测试场景：简单对话、单次工具调用、多次工具调用
  - **验收**：测试通过，能在代码中调用 Agent

### Sprint 2: CLI 交互（Day 3）

- [ ] **2.1 实现 REPL 基础**
  - 文件：`packages/agent/src/cli/repl.ts`
  - 使用 readline 读取用户输入
  - 调用 Agent Loop 处理
  - 显示响应
  - 支持退出命令（exit, quit, Ctrl+C）
  - **验收**：能启动交互式会话

- [ ] **2.2 实现输出渲染**
  - 文件：`packages/agent/src/cli/renderer.ts`
  - 使用 chalk 添加颜色
  - 显示工具调用过程（工具名 + 参数摘要）
  - 显示工具结果（成功/失败）
  - 显示 token 使用量
  - **验收**：输出美观易读

- [ ] **2.3 实现 CLI 入口**
  - 文件：`packages/agent/src/cli/index.ts`
  - 解析命令行参数
  - 加载环境变量
  - 启动 REPL
  - **验收**：`pnpm cli` 能启动

- [ ] **2.4 Checkpoint 2 - CLI 验证**
  - 手动测试 4 个核心场景
  - 修复发现的问题
  - **验收**：4 个场景都能正常工作

### Sprint 3: 完善和测试（Day 4-5）

- [ ] **3.1 错误处理完善**
  - LLM 调用失败重试（3 次，指数退避）
  - 工具执行错误友好提示
  - 网络超时处理
  - **验收**：错误场景有友好提示

- [ ] **3.2 工具调用优化**
  - 检查内置工具是否都能正常工作
  - 修复工具执行问题
  - 添加工具执行超时
  - **验收**：6 个内置工具都能正常使用

- [ ] **3.3 集成测试**
  - 创建 `packages/agent/test/integration/agent-loop.test.ts`
  - 使用真实 API 测试完整流程
  - 测试各种场景
  - **验收**：集成测试通过

- [ ] **3.4 文档更新**
  - 更新 README.md 使用说明
  - 添加环境变量配置说明
  - 添加快速开始指南
  - **验收**：新用户能按文档跑起来

### Sprint 4: 发布准备（Day 6-7）

- [ ] **4.1 代码清理**
  - 删除无用代码
  - 统一代码风格
  - 添加必要注释
  - **验收**：代码整洁

- [ ] **4.2 构建和打包**
  - 确保 `pnpm build` 成功
  - 确保 `pnpm cli` 能运行打包后的代码
  - **验收**：构建成功

- [ ] **4.3 MVP 完成报告**
  - 总结实现的功能
  - 记录已知问题
  - 规划下一步迭代
  - **验收**：报告完成

## 注意事项

1. **复用已有代码**
   - Tool 定义和 Registry 已经实现，直接用
   - 内置工具已经实现，检查是否能用
   - Session 模块已有，MVP 可简化使用

2. **不要过度设计**
   - MVP 不需要 MCP 集成
   - MVP 不需要流式输出
   - MVP 不需要复杂的权限控制

3. **快速验证**
   - 每完成一个任务就手动测试
   - 发现问题立即修复
   - 不要积累技术债务

4. **保持简单**
   - 代码能工作就行，不追求完美
   - 后续迭代再优化
